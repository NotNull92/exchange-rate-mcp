## 🎮 Prologue : 프레임 드랍을 참을 수 없는 뉴비 (직접 스크립트를 짜야 하는 이유)

요즘 AI 씬이 핫하다. 코드 짜주고 버그 픽스까지 알아서 싹 해주는 갓겜 메타라고 난리다.
하지만 막상 에디터를 켜고 "지금 1달러가 얼마야?" 하고 물어보면, 이 녀석들은 하드코딩된 과거 데이터만 앵무새처럼 읊어댄다. 브라우징 플러그인을 붙여줘도 씬 로딩 오지게 길고 GC 스파이크 튀듯이 버벅거린다.

"아니 최첨단 인공지능이라며! 왜 런타임 실시간 데이터 하나 못 가져와서 뉴비 티를 내는 건데?!"

그래서 알아봤다. AI 코어 엔진에 내가 직접 스크립트 컴포넌트를 붙여줄 수 있는 궁극의 플러그인 아키텍처. 바로 **MCP(Model Context Protocol)** 였다. 이걸 쓰면 내 로컬 망에 띄워둔 커스텀 API 컴포넌트를 AI 인벤토리에 강제로 우겨넣을 수 있다고 한다. 

혼자 멘헤라 올 것 같아서, 코딩 파트너이자 유능한 AI 조수 **Antigravity**를 소환했다.
"Antigravity야, 나랑 같이 실시간 환율 조회 에셋 하나만 깎아보자."

---

## ⚔️ Scene 1 : 패키지 매니저 세팅 및 무료(?) 에셋 임포트

일단 국룰인 TypeScript로 커스텀 MonoBehaviour 급의 기본 뼈대 스크립트를 짰다. `@modelcontextprotocol/sdk` 패키지를 임포트하고, `get_exchange_rate` 라는 멋들어진 이름의 public 메서드를 선언해줬다. 

데이터 소스가 필요하니 에셋 스토어를 뒤져 `exchangerate-api.com`을 찾았다. 회원가입하고 API 키를 받아왔다.
당연히 깃허브 퍼블릭 레포에 푸시를 하면 봇들이 1초 만에 크롤링해서 내 API 한도를 박살낼 테니, `.env` 파일이라는 안전 지대로 키를 옮겼다.
그리고 dotenv 패키지를 씬 하이어라키에 추가했다.

```typescript
import dotenv from "dotenv";
dotenv.config();

const API_KEY = process.env.EXCHANGERATE_API_KEY; // 보안은 생명이다
```

빌드 성공. 0 Errors. 이때까지만 해도 난 내가 시니어 테크니컬 아티스트 쯤 되는 줄 알았다.

---

## 💀 Scene 2 : 널포인터 익셉션 출몰 - JSON-RPC 파괴자 등장

이제 설정 파일에 내 로컬 서버 진입점을 연결해줬다. 에디터 플레이 버튼을 누르고 경건한 마음으로 Antigravity에게 프롬프트를 날렸다.
"환율 가져와 벅벅"

하지만 콘솔창은 붉게 물들었다.
**"[Error] 유효하지 않은 JSON-RPC 통신. 파싱 실패."**

"어라? 내 스크립트는 컴파일 에러가 없는데??"
디버거를 켜고 메모리를 트레이싱한 지 수십 분... 범인은 다름 아닌 눈치 없는 dotenv 녀석의 Awake 방식에 있었다. 
MCP 서버는 기본적으로 AI 클라이언트와 터미널 표준 입출력이라는 좁은 스트림 채널로만 JSON-RPC 객체를 직렬화해서 주고받는다. 
그런데 이 핑프 같은 dotenv가 런타임에 올라가자마자 "나 환경변수 다 읽어옴ㅋ" 하고 잡다한 디버그 로그를 그 채널에 냅다 쏴버린 것이다!
그 결과, 깔끔한 JSON 데이터를 기다리고 있던 파서에 기괴한 문자열 찌꺼기가 쑤셔박혔고, 결국 파싱 에러를 토해버린 것.

**[핫픽스]** 쿨하게 패키지 매니저에서 dotenv를 우클릭 후 Remove 해버렸다. 설정 파일 env 딕셔너리에 직접 환경변수를 주입할 수 있다는 갓팁을 공식 문서 끄트머리에서 발견했기 때문. 엔진 최적화 1승 적립.

## 🔥 Scene 3 : 기획 팀의 태클 (서버 틱레이트 왜 이래요?)

API 키를 런타임에 주입하고 씬을 돌리니 드디어 값이 출력된다! 프레임 드랍 없이 깔끔하다.
...근데 잠깐, 이거 어제 환율 픽스본 아니야?

알고 보니 내가 쓰던 무료 티어 에셋은 **24시간에 한 번씩만 코루틴이 도는** 끔찍한 폴링 API였다. 
실시간 환율 서버(가짜)를 만들 바엔 이 프로젝트 드랍하고 만다. 게다가 나 혼자 쓸 거면 env 환경변수를 쓰든 말든 상관없지만, 남들 쓰라고 리드미에 "API 키 발급받아서 환경변수 세팅하고 빌드하세요^^" 라고 적으면 뉴비 절단기 소리 듣기 딱 좋지 않은가?

다시 에셋 스토어 폭풍 검색 모드 돌입.
그리고 찾아낸 궁극의 무료 에셋, **Frankfurter API**.
유럽 중앙은행(ECB) 공식 데이터를 기반으로 제공되며, **회원가입? API 키? 과금 제한? 그딴 거 없다.** 그냥 완전 오픈소스 에셋이다. 심지어 매 영업일 서버타임 오후 4시마다 꼬박꼬박 핑 갱신이 된다!!

즉시 API 키 예외 처리 코드 블록을 전부 드래그해서 날려버리고, 네트워크 리퀘스트 메서드를 교체했다.

```typescript
// 굿바이 토큰 인증 라이프사이클
const response = await axios.get(`https://api.frankfurter.app/latest?from=${fromCurrency}&to=${toCurrency}`);
```

## 🌪️ Scene 4 : 하드코딩의 폐해 (로컬라이제이션 이슈)

이제 완벽한 글로벌 원빌드 에셋이 탄생할 줄 알았던 찰나, QA 도중 치명적인 로직 버그가 리포트되었다. 
유럽 중앙은행 형님들이라 그런지 자기들 밸런싱 기준표에 있는 주요 30개국 화폐가 아니면 서버 측에서 가차 없이 **404 에러**를 때려버리는 것이었다.
대만 달러(TWD), 베트남 동(VND), 러시아 루블(RUB)을 파라미터로 넘기면 통신이 대차게 터졌다.

"글로벌 런칭 에셋이라면서 리전 포팅이 이따위냐!"

어쩔 수 없다. 게임 개발자 특기인 **"폴백 방어코드 떡칠"** 을 시전한다. 
예외를 뱉고 크래시 나는 대신, **이중 API 연결망** 시스템을 아키텍처 레벨에 도입했다.
메인 노드인 Frankfurter가 모른다고 익셉션을 던지면, catch 문에서 재빨리 150여 개국 지원이 되지만 묘하게 레이턴시가 튀는 서브 노드 jsdelivr currency-api로 비동기 호출을 다시 쏘게 만들었다.

```typescript
try {
   // 일단 1티어 랭커(유럽 중앙은행 형님들)한테 Request ㄱㄱ
   const res = await RequestPrimaryServer(...);
} catch (error) {
   // 형님들이 취급 안한다고 하면 중립 몹(jsdelivr CDN) 스폰시켜서 파밍
   const fallbackRes = await RequestFallbackServer(...); 
}
```

결과는? **전 세계 거의 모든 빌드 타겟 커버 완성!** 이제 어떤 뉴비가 와서 "태국 바트에서 짐바브웨 달러 얼만지 렌더링 좀?" 해도 절대 크래시 안 나고 콜백을 리턴해준다.

## ⚔️ Final Boss : NPM 릴리즈 퍼블리싱 (퍼블리셔의 횡포)

이제 내가 Antigravity와 함께 영혼을 갈아넣은 빛나는 마스터 빌드를 NPM에 배포할 시간이다. 
누구나 CLI 커맨드 한 줄 `npx -y exchange-rate-mcp` 이면 종속성 꼬임 방지 샌드박스 위에서 이 서버 로직을 돌릴 수 있게 만들고 싶었다.

패기롭게 터미널에 `npm publish` 치트키를 입력했다.

1. **빌드 에러 1: Invalid Entry Point (너의 bin 경로는 유효하지 않다.)**
   - package.json 매니페스트에서 `"bin": { "exchange-rate-mcp": "./dist/index.js" }` 라고 썼는데, 앞에 상대 경로 점을 찍었다고 패키징 밴을 먹었다. `./`를 빼고 `dist/index.js`로 수정하여 프리컴파일 경로 지정 완성.
2. **빌드 에러 2: Missing Execution Context (나를 뭘로 어떤 엔진에서 렌더링하라고?)**
   - 글로벌 터미널 런타임에서 이 패키지를 단독 실행하려면 OS가 이 파일을 C++ 런타임으로 돌릴지 Node.js V8 엔진으로 돌릴지 알아야 한다. 눈물을 머금고 빌드 원본 엔트리 파일 최상단에 `#!/usr/bin/env node` 라는 괴상한 마법의 주문(Shebang 플래그)을 베이크 해줬다.
3. **빌드 에러 3: 403 Forbidden - Security Policy (2단계 인증 안 하면 못 지나간다 애송이.)**
   - npm 회원가입만 하면 자동 승인인 줄 알았더니, 요즘 씬이 흉흉하다고 무조건 2단계 인증(OTP)을 뚫어야 배포 파이프라인을 열어준단다. 핸드폰 꺼내서 핑거 스냅으로 랜덤 시드값 6자리를 입력했다.

**그리고 마침내 콘솔에 뜬 초록색 정상 렌더링 텍스트...**
`+ exchange-rate-mcp@1.0.0`

**NPM 공식 에셋 스토어 등재 (Release v1.0.0) 골드행 성공!** 🎉🎉🎉
